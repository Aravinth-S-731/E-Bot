WORKING

User input

Let's look at how the questions posed by users are treated.

User query

How can i pay my electric bill ?

Steps involved

Step-1

Tokenizing the sentence

In Natural Language Processing, tokenization divides a string into a list of tokens. Tokens come in handy when finding valuable patterns and helping to replace sensitive data components with non-sensitive ones. word_tokenize is a function in Python that splits a given sentence into words using the NLTK library. Then each of the words are converted into lowercase.


How can i pay my electric bill ?

↓

['how','can','i','pay','my','electric','bill','?']

Step-2

BoW (Bag of Words)

Using BoW, we can calculate the repeated words in the sentence if present and prioritize the word using boolean values 1 (if repeated) or 0 (not repeated). Then this bag of words are cleaned up using NLTK packages by comparing it with the predefined set of word such as 'a','the','is'. It also removes special characters.


['how','can','i','pay','my','electric','bill','?']

↓

['how','can','i','pay','my','electric','bill','?']

Step-3

Lemmatizing the words

Lemmatization is the process of grouping together the different inflected forms of a word so they can be analyzed as a single item. Lemmatization is similar to stemming but it brings context to the words. So it links words with similar meanings to one word. Example, the word 'better' â‡› 'good'.


pay → pay

electric→ electricity

bill → bill

Step-4

Matching the lemmatized words

After lemmatizing the words, the words are compared with the pre-defined set of answers. If the lemmatized keyword matches with the tag of the answer, the data is fetched from the .json file.
